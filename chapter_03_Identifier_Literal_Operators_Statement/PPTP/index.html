19 - Identifiers (Reserved Keywords)

19_Identifier.js Identifiers

Demonstrates that reserved keywords like class, if, and return cannot be used as identifiers. Also warns about using non-reserved but built-in names like undefined and Infinity as identifiers -- technically allowed but extremely dangerous.

// let class = "Hello World";
// // class is a reserved keyword in JavaScript, so it cannot be used as an identifier.
// let if = true;
// // if is also a reserved keyword in JavaScript.

// let return = 5;
// // return is also a reserved keyword in JavaScript.

// let myVariable = 10;
// // This is a valid identifier and can be used without any issues.

// // they look like keywords but aren't reserved)
// let undefined = 5;  // Works but NEVER do this!
// let Infinity = 10; // Works but NEVER do this!

20 - Literals Introduction

20_literla.js Literals

Introduces the concept of literals -- fixed values written directly in your code. Covers string literals, boolean literals, numeric literals, null, and undefined.

let age = "pramod"; // This is a string literal
let isStudent = true; // This is a boolean literal
let pi = 3.14; // This is a numeric literal
let name = 'Alice'; // This is a string literal
let nullValue = null; // This is a null literal
let undefinedValue; // declared but not assigned, value is 'undefined'

23 - null vs undefined Comparisons

23_null_undefined.js Equality

Deep comparison between null and undefined using strict (===) and loose (==) equality. Shows that null === undefined is false (different types), while null == undefined is true. Neither equals 0 or empty string with loose equality.

console.log(null === undefined); // (strict — different types)
console.log(null == undefined);

console.log(null == 0);
console.log(null == "");
console.log(undefined == 0);
console.log(undefined == "");

24 - == vs === (Equality Operators)

24_equla_triequal.js Operators

Explains the three equals: = (assignment), == (loose equality with type coercion), and === (strict equality checking both type and value). Shows that 5 == "5" is true but 5 === "5" is false.

// ==, ===, =

let a = 5;

console.log(5 == "5");
console.log(5 === "5");  // number, string

console.log(null == undefined); // special rule, they only equal each other
console.log(null === undefined);

console.log(5 == 5.0);
console.log(5 === 5.0);
console.log(5 === 5.01);
